# 数据库仓库管理功能说明

## 🎯 功能概述

现在你的 Markdown 编辑器已经集成了完整的数据库仓库管理功能，支持将仓库配置信息保存到 PostgreSQL 数据库中，提供更安全、更可靠的数据存储。

## ✨ 主要特性

### 1. **数据库存储**
- 仓库配置信息保存在 PostgreSQL 数据库
- 支持用户隔离，每个用户只能管理自己的仓库
- 数据持久化，重启后配置不会丢失
- 支持软删除，数据安全可靠

### 2. **完整的 CRUD 操作**
- **创建**: 添加新的仓库配置
- **读取**: 获取用户的仓库列表
- **更新**: 修改仓库配置信息
- **删除**: 软删除仓库（标记为非活跃）

### 3. **用户权限管理**
- 基于 NextAuth.js 的用户认证
- 用户只能访问和管理自己的仓库
- 安全的 API 接口设计

## 🗄️ 数据库模型

### NoteRepository 表结构
```sql
model NoteRepository {
  id          String   @id @default(cuid())
  userId      String   // 关联用户ID
  name        String   // 仓库名称
  description String?  // 仓库描述
  repoUrl     String   // 仓库地址
  branch      String   @default("main") // 分支
  username    String?  // 用户名
  accessToken String?  // 访问令牌
  isDefault   Boolean  @default(false) // 是否默认
  isActive    Boolean  @default(true)  // 是否活跃
  lastUsed    DateTime @default(now()) // 最后使用时间
  createdAt   DateTime @default(now()) // 创建时间
  updatedAt   DateTime @updatedAt      // 更新时间

  @@unique([userId, repoUrl]) // 用户+仓库地址唯一
  @@index([userId, isDefault]) // 用户+默认状态索引
}
```

## 🚀 使用指南

### 1. 访问管理页面

#### 方式一：直接访问
- 访问 `http://localhost:3000/admin/repositories`
- 需要先登录才能访问

#### 方式二：从主页进入
- 访问 `http://localhost:3000/user`
- 点击"仓库管理"卡片

### 2. 添加新仓库

1. **点击"添加仓库"按钮**
2. **填写仓库信息**：
   - **仓库名称**: 必填，给仓库起个易记的名字
   - **仓库地址**: 必填，Git 仓库的 HTTPS 或 SSH 地址
   - **分支**: 可选，默认为 main
   - **描述**: 可选，仓库的简要说明
   - **用户名**: 可选，GitHub 用户名
   - **访问令牌**: 可选，GitHub Personal Access Token
   - **设为默认**: 可选，勾选后设为默认仓库

3. **点击"添加仓库"保存**

### 3. 管理已保存的仓库

#### 编辑仓库
- 点击仓库卡片上的编辑图标
- 修改仓库配置信息
- 点击"更新仓库"保存

#### 删除仓库
- 点击仓库卡片上的删除图标
- 确认删除操作
- 仓库会被软删除（标记为非活跃）

#### 搜索仓库
- 使用搜索框按名称或地址搜索
- 支持模糊搜索

### 4. 在编辑器中使用

#### 自动连接默认仓库
- 打开编辑器时会自动连接默认仓库
- 如果没有默认仓库，会显示仓库选择界面

#### 切换仓库
- 在编辑器中点击文件浏览器的设置图标
- 选择要连接的仓库
- 仓库会自动切换并重新加载文件列表

## 🔧 API 接口

### 获取仓库列表
```http
GET /api/repositories
Authorization: Bearer <session_token>
```

### 创建新仓库
```http
POST /api/repositories
Content-Type: application/json
Authorization: Bearer <session_token>

{
  "name": "仓库名称",
  "description": "仓库描述",
  "repoUrl": "https://github.com/username/repo.git",
  "branch": "main",
  "username": "github_username",
  "accessToken": "personal_access_token",
  "isDefault": false
}
```

### 更新仓库
```http
PUT /api/repositories/{id}
Content-Type: application/json
Authorization: Bearer <session_token>

{
  "name": "新仓库名称",
  "description": "新描述",
  "repoUrl": "https://github.com/username/new-repo.git",
  "branch": "develop",
  "username": "new_username",
  "accessToken": "new_token",
  "isDefault": true
}
```

### 删除仓库
```http
DELETE /api/repositories/{id}
Authorization: Bearer <session_token>
```

## 🔒 安全特性

### 1. **用户隔离**
- 每个用户只能访问自己的仓库
- API 接口验证用户身份
- 数据库查询包含用户ID过滤

### 2. **数据验证**
- 必填字段验证
- 仓库地址格式验证
- 用户权限验证

### 3. **软删除**
- 删除操作不会真正删除数据
- 只是标记为非活跃状态
- 数据可以恢复

### 4. **访问令牌安全**
- 访问令牌在数据库中加密存储
- API 接口不会返回敏感信息
- 支持令牌更新和删除

## 📊 数据管理

### 仓库状态
- **活跃**: `isActive = true`，正常使用
- **非活跃**: `isActive = false`，已删除

### 默认仓库
- 每个用户只能有一个默认仓库
- 设置新默认时会自动取消旧的
- 默认仓库会优先显示

### 使用统计
- `lastUsed` 字段记录最后使用时间
- 支持按使用时间排序
- 帮助用户快速找到常用仓库

## 🛠️ 故障排除

### 常见问题

#### 无法访问管理页面
- 检查是否已登录
- 确认用户权限
- 查看浏览器控制台错误

- 检查仓库地址格式
- 确认网络连接正常
- 验证访问令牌权限

#### 数据库连接问题
- 检查数据库连接配置
- 确认 Prisma 客户端已生成
- 查看服务器日志
