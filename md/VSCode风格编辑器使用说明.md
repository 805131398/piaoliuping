# VSCode 风格 Markdown 编辑器使用说明

## 🎯 新功能概述

现在你的 Markdown 编辑器已经升级为类似 VSCode 的使用体验，包含以下核心功能：

### ✨ 主要特性

1. **仓库管理系统**
   - 保存多个 Git 仓库配置
   - 设置默认仓库
   - 快速切换仓库
   - 仓库信息持久化存储

2. **VSCode 风格界面**
   - 目录式文件浏览器
   - 文件夹展开/折叠
   - 文件搜索功能
   - 当前文件状态显示

3. **智能文件管理**
   - 自动打开选中的文件
   - 文件修改状态跟踪
   - 实时预览编辑
   - 安全的文件路径验证

## 🚀 使用指南

### 1. 仓库管理

#### 添加新仓库
1. 访问 `/md-editor` 页面
2. 点击"选择仓库"卡片中的设置图标
3. 在仓库管理对话框中点击"添加仓库"
4. 填写仓库信息：
   - **仓库名称**: 给仓库起个易记的名字
   - **仓库地址**: Git 仓库的 HTTPS 或 SSH 地址
   - **分支**: 要编辑的分支（默认为 main）
   - **用户名**: GitHub 用户名（可选）
   - **访问令牌**: GitHub Personal Access Token（可选，私有仓库需要）
   - **设为默认仓库**: 勾选后会自动连接此仓库

#### 管理已保存的仓库
- **切换仓库**: 点击仓库卡片上的连接图标
- **编辑仓库**: 点击编辑图标修改仓库信息
- **删除仓库**: 点击删除图标移除仓库
- **设置默认**: 在编辑时勾选"设为默认仓库"

### 2. 文件浏览器

#### 目录式浏览
- **展开文件夹**: 点击文件夹前的箭头或文件夹名称
- **折叠文件夹**: 再次点击已展开的文件夹
- **选择文件**: 点击文件名称打开编辑

#### 搜索功能
- 在搜索框中输入文件名或路径
- 支持模糊搜索
- 实时过滤显示结果

#### 文件状态
- **当前文件**: 高亮显示正在编辑的文件
- **文件类型**: 不同图标表示文件夹、Markdown 文件等
- **状态栏**: 显示当前选中的文件路径

### 3. 编辑器功能

#### 自动打开文件
- 点击文件后自动在编辑器中打开
- 支持实时预览
- 语法高亮和自动补全

#### 文件操作
- **保存**: 点击保存按钮或使用快捷键
- **修改状态**: 文件修改后会显示"已修改"标识
- **Git 操作**: 支持提交、推送、拉取等

## 🔧 高级功能

### 仓库管理
- **持久化存储**: 仓库配置保存在本地，重启后仍然可用
- **默认仓库**: 设置默认仓库后，打开编辑器会自动连接
- **多仓库支持**: 可以保存多个仓库，快速切换

### 文件管理
- **安全验证**: 防止目录遍历攻击
- **路径验证**: 确保文件路径在仓库目录内
- **错误处理**: 完善的错误提示和调试信息

### Git 集成
- **状态检查**: 实时查看 Git 状态
- **提交管理**: 支持自定义提交信息
- **分支操作**: 支持切换分支和拉取更新

## 📋 快捷键和操作

### 文件操作
- **打开文件**: 点击文件名称
- **搜索文件**: Ctrl/Cmd + F 或点击搜索框
- **切换仓库**: 点击设置图标打开仓库管理

### 编辑器操作
- **保存文件**: Ctrl/Cmd + S
- **实时预览**: 自动显示在右侧预览区
- **语法高亮**: 自动识别 Markdown 语法

## 🛠️ 故障排除

### 常见问题

#### 无法连接仓库
- 检查仓库地址是否正确
- 确认网络连接正常
- 验证访问权限和令牌

#### 文件无法打开
- 检查文件路径是否正确
- 确认文件存在于仓库中
- 查看浏览器控制台错误信息

#### 仓库管理问题
- 清除浏览器缓存
- 检查本地存储权限
- 重新添加仓库配置

### 调试信息
- 打开浏览器开发者工具
- 查看控制台日志信息
- 检查网络请求状态

## 🎯 最佳实践

### 仓库管理
1. **命名规范**: 使用有意义的仓库名称
2. **默认设置**: 为常用仓库设置默认
3. **权限管理**: 妥善保管访问令牌

### 文件组织
1. **目录结构**: 使用清晰的文件夹结构
2. **命名规范**: 使用描述性的文件名
3. **版本控制**: 定期提交和推送更改

### 工作流程
1. **连接仓库**: 选择或添加要编辑的仓库
2. **浏览文件**: 使用文件浏览器找到目标文件
3. **编辑文件**: 在编辑器中修改内容
4. **保存更改**: 保存文件到本地
5. **提交推送**: 使用 Git 操作提交和推送更改
